# Architecture

## Overview

Mongol Beauty is a full-stack e-commerce monorepo. The frontend is a React SPA served by nginx. The backend is a NestJS GraphQL API backed by PostgreSQL. In production all services run in Docker containers coordinated by Docker Compose.

## Frontend

### Routing

- Entry: `apps/web/src/main.tsx` → `<App>` → `<AppRoutes>` wrapped in `<ErrorBoundary>`
- Routes defined in `apps/web/src/app/routes.tsx`; all pages lazy-loaded via `app/lazy-pages.ts`
- Storefront under `<MainLayout>` at `/`
- Admin panel under `<AdminLayout>` + `<ProtectedAdminRoute>` at `/admin`
- Admin login at `/admin/login` — uses a separate `useAdminAuth()` hook with its own token

### Apollo Client

- Client instantiation: `apps/web/src/graphql/apollo-client.ts`
- Link chain: `apps/web/src/graphql/apollo-links.ts` (auth header injection → error handling → HTTP link)
- Cache field policies: `apps/web/src/graphql/apollo-cache.ts`
- All GraphQL operations are in typed files under `apps/web/src/graphql/`

### GraphQL Codegen

```
apps/api/src/schema.gql          ← auto-generated by NestJS on API start
  ↓  (copy manually after schema changes)
apps/web/src/graphql/schema.graphql   ← hand-maintained static copy
  ↓  yarn graphql:codegen
apps/web/src/graphql/generated/graphql.ts   ← typed hooks and operations
```

After any API schema change: copy `apps/api/src/schema.gql` to `apps/web/src/graphql/schema.graphql`, then run `yarn graphql:codegen`. CI fails on drift.

### Pricing and Discounts

Product price discounts are applied in `product.resolver.ts` based on the authenticated user's `userType`:

| User type | Discount |
|-----------|----------|
| `USER` (default) | None — full price |
| `SUBSCRIBED_USER` | 10% off (`price * 0.9`) |
| `VIP_USER` | 20% off (`price * 0.8`) |
| `ADMIN` | No discount (admin views raw price) |

`discountedPrice` is a nullable computed field. When `null`, the frontend shows only `price`.

## Backend

### Service Modes (`SERVICE_MODE` env)

The same compiled API image runs in three different modes depending on `SERVICE_MODE`:

| Mode | Default port | Loads |
|------|-------------|-------|
| `gateway` (default) | 4000 | All modules — GraphQL API, orchestrates order + payment |
| `order` | 4010 | OrderModule only — REST endpoints |
| `payment` | 4020 | OrderModule + PaymentModule — REST endpoints |

The branching is in `apps/api/src/app.module.ts` via `isGateway` / `isOrderService` / `isPaymentService` flags.

In local dev (`yarn dev`) all three modes collapse into a single in-process call with no HTTP hop between them.

### Auth

- **Admin**: `adminLogin` mutation → JWT with `{ isAdmin: true }` claim → `GqlAdminGuard`
- **Storefront user**: `userLogin` mutation → JWT with `{ userType }` claim → `GqlJwtAuthGuard`
- **Optional auth**: `GqlOptionalAuthGuard` — resolves user if token present, continues as guest if not
- Tokens stored in localStorage on the client; sent as `Authorization: Bearer <token>` header
- Auth cookies (`httpOnly`, `secure`, `sameSite: none` in production) are also set as a fallback

### Guards

| Guard | Purpose |
|-------|---------|
| `GqlOptionalAuthGuard` | Attaches user to context if valid token; does not block unauthenticated requests |
| `GqlJwtAuthGuard` | Blocks unauthenticated requests with 401 |
| `GqlAdminGuard` | Blocks non-admin users (requires `GqlJwtAuthGuard` first) |

### DataLoader

`DataloaderModule` provides `ProductLoader` and `CategoryLoader` to batch and cache DB lookups within a single GraphQL request. Both are injected into the GraphQL context as `ctx.loaders.product` and `ctx.loaders.category`.

### Rate Limiting

`ThrottlerModule` with `GqlThrottlerGuard` applied globally. Tight limits on auth endpoints:
- Admin login: 10 req / 60 s
- User login: 15 req / 60 s
- Register: 5 req / 60 s
- Forgot password: 5 req / 60 s

GraphQL query depth limit: 10.

## Database

PostgreSQL 15. TypeORM code-first with entities in `*.entity.ts`.

### Key tables

| Table | Description |
|-------|-------------|
| `users` | All users: admins, subscribers, VIPs, regular |
| `products` | Products with price, stock, skin type tags, images |
| `categories` | Product categories with slug and image |
| `orders` | Customer orders with status state machine |
| `order_items` | Line items per order (product snapshot at order time) |
| `cart_items` | Persistent cart per user |
| `site_settings` | Delivery fee, free delivery threshold, bank info (singleton row) |
| `blog_posts` | CMS content pages |
| `vouchers` | Discount voucher codes |

### Migrations

Migrations live in `apps/api/src/migrations/`. In development, `DB_SYNCHRONIZE=true` (default) auto-syncs schema from entities. In production, the `migrator` container runs `migration:run` before the gateway starts, and `DB_SYNCHRONIZE=false`.

## Order Flow

```
Browse products
  → Add to cart (cart_items, per-user DB-backed)
    → Create order (order + order_items created, status: WAITING_PAYMENT)
      → Customer uploads bank transfer receipt (payment-service → Cloudflare R2)
        → Admin reviews receipt in /admin/orders
          → Admin confirms payment (status: PAID_CONFIRMED)
            → Admin marks shipped (status: SHIPPING)
              → Admin marks complete (status: COMPLETED)
                            ↕ at any point before completion
                       Admin cancels (status: CANCELLED)
```

`PaymentMethod` enum: `BANK_TRANSFER` (receipt required) | `CASH`.

## Production Infrastructure

```
Internet
  └── nginx-proxy (ports 80/443, Let's Encrypt TLS via acme-companion)
        ├── mcosmetics.mn        → mb-web (nginx static + /graphql proxy)
        │                              └── /graphql → mb-gateway:4000
        └── [api subdomain]      → mb-gateway:4000 (optional, same container)

Docker Compose services:
  mb-postgres       PostgreSQL 15
  mb-migrator       Runs migrations then exits
  mb-order-service  REST order service (:4010)
  mb-payment-service REST payment service (:4020)
  mb-gateway        NestJS GraphQL gateway (:4000)
  mb-web            nginx serving React build
  mb-nginx-proxy    Reverse proxy + TLS termination
  mb-acme           Let's Encrypt cert manager
  mb-db-backup      Daily pg_dump to host directory
```

All containers share `mongol-beauty-prod-network`. The `mb-web` nginx config proxies `/graphql` and `/uploads` to `mb-gateway` by container name.
